## Описание файлов проекта

В репозитории содержатся только исходный код и конфигурационные файлы проекта. Тяжелые бинарные данные (веса моделей `.pt` и тестовые видео) исключены из репозитория для оптимизации размера.

* **`main.py`** — основной и единственный исполняемый файл приложения. Содержит в себе:
  * Графический интерфейс оператора (построен на `Tkinter` и `Pillow`).
  * Многопоточный движок (декодирование видеопотока через `OpenCV` вынесено в отдельный поток, чтобы интерфейс не зависал во время инференса нейросети).
  * Математический модуль проверки пересечения bounding box человека с полигонами зон.
  * Логику сбора аппаратных метрик производительности.
* **`train.py`** — скрипт для обучения и тонкой настройки (Fine-Tuning) нейросети YOLO под конкретный датасет. Содержит в себе:
  * Автоматическую генерацию уникального имени папки обучения по дате и времени (защита от перезаписи результатов).
  * Настройки аппаратного ускорения (принудительное использование GPU и автоматический расчет ядер CPU для быстрой загрузки данных).
  * Фильтрацию классов (`classes=[0]`) для фокусировки модели исключительно на распознавании людей.
  * Логику автоматического поиска и вывода финального пути к лучшим полученным весам (`best.pt`).
* **`data.yaml`** — конфигурационный файл датасета. Содержит в себе абсолютные пути к обучающей и валидационной выборкам снимков, а также имя и идентификатор целевого класса (`0: person`).
* **`.gitignore`** — служебный файл конфигурации Git. Он автоматически блокирует случайную загрузку в облако тяжелых файлов: весов нейросетей (`.pt`), видеороликов (`.mp4`, `.avi`), скриншотов нарушений и кэша Python.
* **`README.md`** — документация проекта (инструкция по запуску, которую вы сейчас читаете).

## Требования к системе

Для запуска проекта на компьютере должны быть установлены:
1. **Python 3.10 / 3.11**
2. **Библиотеки зависимостей:**
   * `ultralytics` (нейросеть YOLO)
   * `opencv-python` (обработка видео и графики)
   * `pillow` (вывод изображений в GUI)
   * `torch` / `torchvision` (желательно с поддержкой CUDA для аппаратного ускорения на GPU)
